# 🚀 AI 投诉处理助手 - 5 分钟极速部署指南

> **目标**: 将 v3.0 旗舰版部署到 Streamlit Cloud
> **耗时**: 5-10 分钟
> **成本**: $0 (免费额度)

---

## 第一步：创建 GitHub 仓库 (2 分钟)

1. 打开 GitHub: https://github.com/new
2. **Repository name**: `ai-complaint-handler`
3. **Public/Private**: 选择 Public (免费) 或 Private (需付费或申请教育包)
4. 点击 **Create repository**

## 第二步：推送代码到 GitHub (2 分钟)

在终端执行以下命令（替换为你的 GitHub 用户名）：

```bash
cd /Users/wtueeq/.hermes/deployments/ai-complaint-handler

# 设置远程仓库地址 (替换 YOUR_GITHUB_USERNAME 为你的 GitHub 用户名)
git remote set-url origin https://github.com/YOUR_GITHUB_USERNAME/ai-complaint-handler.git

# 推送代码
git push -u origin main
```

如果遇到认证问题，使用 Personal Access Token：
1. 访问：https://github.com/settings/tokens
2. 创建新 Token (勾选 `repo` 权限)
3. 推送时使用 Token 代替密码

## 第三步：在 Streamlit Cloud 部署 (3 分钟)

1. **访问**: https://share.streamlit.io
2. **登录**: 使用 GitHub 账号登录
3. **点击**: "New app" 或 "Add an app"
4. **配置**:
   - **Git repository**: 选择 `ai-complaint-handler`
   - **Branch**: `main`
   - **Main file path**: `ai_complaint_handler_v3.py`
   - **Advanced settings** (可选):
     - 添加环境变量 `MINIMAX_API_KEY` (如果有)
5. **点击**: "Deploy!"

## 第四步：等待部署完成 (2-5 分钟)

- Streamlit Cloud 会自动安装依赖 (`requirements.txt`)
- 构建完成后，你会看到一个绿色的 "Ready" 状态
- 点击 **Open app** 查看你的应用

**部署成功后的 URL 格式**:
```
https://YOUR_GITHUB_USERNAME-ai-complaint-handler-xxxxxx.streamlit.app
```

## 常见问题解决

### 问题 1: 推送失败 (Authentication failed)
**解决**: 使用 Personal Access Token 代替密码
```bash
git remote set-url origin https://YOUR_USERNAME:YOUR_TOKEN@github.com/YOUR_USERNAME/ai-complaint-handler.git
```

### 问题 2: 部署失败 (Module not found)
**解决**: 确保 `requirements.txt` 包含所有依赖
```txt
streamlit==1.50.0
pandas
requests
openpyxl
```

### 问题 3: API Key 缺失
**现象**: 提示 "MINIMAX_API_KEY not found"
**解决**: 
- 方案 A: 在 Streamlit Cloud 后台添加 Secret `MINIMAX_API_KEY`
- 方案 B: 使用模拟模式 (默认降级，无需 API Key)

## 自定义域名 (可选)

如果拥有 yupeng-ai.cn 域名：
1. 在 Streamlit Cloud 设置中找到 "Custom Domain"
2. 按照指引配置 DNS 记录
3. 绑定域名：`complaint.yupeng-ai.cn`
